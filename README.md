# Taj Dental Center – نظام إدارة عيادة أسنان متكامل

نظام ويب كامل لإدارة عيادة طب الفم والأسنان، يشمل موقعاً تفاعلياً للمرضى لحجز المواعيد، ولوحة تحكم إدارية متطورة (للطبيب والمساعد) تشمل إدارة الحجوزات، المرضى، الزيارات، الخدمات، الحسابات اليومية، التقارير، والصلاحيات.

## 🚀 العروض الحية
- الموقع العام: [https://taj-clinic.netlify.app](https://taj-clinic.netlify.app)
- لوحة التحكم: [https://taj-clinic.netlify.app/admin.html](https://taj-clinic.netlify.app/admin.html)

## 💡 نظرة عامة على المشروع
تم تطوير هذا النظام لتلبية احتياجات عيادة أسنان حقيقية (مركز تاج) بأسيوط. يقدم النظام:
- حجز مواعيد ذكي مع التحقق من الأيام المغلقة، الحد الأقصى اليومي، وحالة استقبال الحجوزات.
- لوحة تحكم كاملة بصلاحيات مختلفة (دكتور / مساعد) لإدارة كل جوانب العيادة.
- تسجيل الزيارات مع إمكانية إضافة خدمات متعددة لكل زيارة، وحساب الإيراد تلقائياً.
- تقارير يومية وشهرية، وإدارة مالية كاملة (إيرادات ومصروفات).
- تصميم متجاوب بالكامل (دعم الجوال، التابلت، والحاسوب).

## 🛠️ التقنيات والأدوات المستخدمة
- **HTML5, CSS3 (متغيرات مخصصة)** – بناء هيكل وتصميم مرن.
- **Bootstrap 5 + RTL** – نظام شبكي سريع ودعم اللغة العربية.
- **JavaScript (ES6+)** – كل المنطق التفاعلي (الحجوزات، لوحة التحكم، إدارة الحالات).
- **Supabase** – قاعدة بيانات سحابية (PostgreSQL) + مصادقة جزئية + تخزين.
- **Font Awesome 6** – أيقونات جاهزة.
- **Google Fonts (Cairo + Poppins)** – تحسين القراءة والجمالية.
- **Netlify** – استضافة آلية مع SSL.

## ✨ الميزات الرئيسية

### للعميل (الموقع العام)
- عرض الخدمات ديناميكياً من قاعدة البيانات.
- حجز موعد مع التحقق الفوري من:
  - إغلاق يوم الأحد.
  - الحد الأقصى لعدد الحجوزات يومياً (قابل للتغيير من الأدمن).
  - الأيام المغلقة يدوياً (إجازة طارئة).
  - حالة استقبال الحجوزات (مفتوحة/مغلقة من الإدارة).
- تنقل سلس بين الأقسام.
- خريطة تفاعلية (Google Maps).
- معلومات الاتصال وساعات العمل.

### لوحة التحكم الإدارية (admin.html)
- **مصادقة ثنائية الدور:** دكتور (كلمة: taj558dc) / مساعد (كلمة: tajdc1010).
- **لوحة إحصائيات** تعرض: إجمالي الحجوزات، حجوزات اليوم (مع حالة تحميل منخفض/مرتفع/كامل)، عدد الأيام المغلقة، عدد الخدمات.
- **إدارة الحجوزات:** عرض، بحث، ترشيح بالتاريخ، تغيير حالة الحجز إلى "تم"، ترتيب الحجوزات يدوياً (رفع/خفض)، حذف، إضافة يدوية.
- **إدارة الخدمات:** إضافة، تعديل، حذف، تحديد تصنيف (كشف/جلسة/علاج/أخرى)، سعر، ترتيب.
- **إدارة المرضى والزيارات:**
  - إضافة/تحديث بيانات المريض (اسم، هاتف، ملاحظات).
  - تسجيل زيارة لمريض مع اختيار خدمة، نوع الزيارة، المبلغ، حالة الدفع، ملاحظات.
  - عرض سجل الزيارات مع إمكانية الحذف.
- **الحسابات اليومية:** إضافة حركات مالية (إيراد/مصروف) بتصنيف ومبلغ وملاحظة، مع حذف الحركات.
- **تقرير اليوم:** ملخص لحجوزات اليوم، عدد الكشوف، الجلسات، الإيراد، المصروف، الصافي، مع حفظ التقرير نهائياً في أرشيف يومي.
- **الإيرادات الشهرية** (ظاهرة فقط للدكتور): عرض عدد الزيارات، عدد الكشوف، الإيراد، الصافي لشهر محدد.
- **إدارة الصلاحيات** (ظاهرة فقط للدكتور): عرض حسابات المستخدمين (الدكتور والمساعدين).
- **إعدادات العيادة:** تحديد الحد الأقصى اليومي، فتح/غلق الحجوزات العامة، إغلاق يوم معين (إجازة طارئة).

### قاعدة البيانات (Supabase)
الجداول المستخدمة:
- `bookings` – الحجوزات (مع دعم sort_order و booking_number).
- `services` – الخدمات (الاسم، التصنيف، السعر، الترتيب، الحالة).
- `patients` – المرضى (الاسم، الهاتف، آخر زيارة، عدد الزيارات، ملاحظات).
- `visits` – الزيارات (ربط بالمريض والحجز والخدمة، النوع، المبلغ، حالة الدفع، ملاحظات).
- `visit_items` – تفاصيل الخدمات المتعددة لكل زيارة (الخدمة، الكمية، السعر، الإجمالي).
- `transactions` – الحركات المالية (التاريخ، النوع، التصنيف، المبلغ، ملاحظة، ربط بالزيارة أو الحجز).
- `daily_reports` – تقارير اليوم (محفوظة لأرشفة الإحصائيات).
- `closed_days` – الأيام المغلقة يدوياً.
- `settings` – إعدادات العيادة (is_open, daily_max).
- `profiles` – حسابات المستخدمين (للتوسع المستقبلي).
